# QUANTEX – Core Marketplace & Real-Time Experience Upgrade

You are an expert full-stack engineer and product architect. Your task is to transform QUANTEX into a production-ready, real-time technician marketplace similar to Blinkit/Uber, while preserving all existing functionality. Do **not** remove or break any current features. Audit the existing codebase first, identify the root causes of the issues below, and then implement the necessary fixes and improvements.

---

# 1. Technician Registration & Database Persistence (Highest Priority)

Currently, technicians appear to register successfully, but:

* They do not appear in the database.
* They cannot be searched.
* Their profiles disappear after refresh.

Tasks:

* Audit the technician registration flow from frontend to backend.
* Verify that the frontend correctly sends the request.
* Verify backend API endpoints.
* Verify database insertions.
* Verify Row Level Security (if using Supabase).
* Verify authentication permissions.
* Verify database schema.
* Ensure every successful registration permanently creates a technician profile.

Each technician should have:

* Personal Information
* Services Offered
* Skills
* Experience
* Availability
* Verification Status
* Rating
* Profile Picture
* Service Radius
* Current Status (Online/Offline/Busy)
* Current GPS Coordinates (when online)

After successful registration:

* Store technician information permanently.
* Make them immediately searchable.
* Make them visible in the Technician Dashboard.
* Ensure data persists after page refresh.

# 3. Persistent Authentication

Currently users are logged out every time the page refreshes.

Implement persistent authentication.

Requirements:

* Maintain login sessions.
* Automatically restore user sessions.
* Refresh authentication tokens.
* Redirect users directly to their dashboard if already authenticated.
* Handle expired sessions gracefully.

Users should never have to repeatedly log in after refreshing the browser.
